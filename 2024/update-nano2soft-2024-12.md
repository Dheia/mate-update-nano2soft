## 2024-12-01
**Support Filter Proposals in user front and back and departments and manage shop**

إضافة فلاتر لفلترة المستخدمين والمتاجر حسب عدد البلاغات أو الشكاوى أو المقترحات التي عليها، وذلك بغرض معرفة الأشخاص أو المتاجر التي عليها بلاغات كثيرة وغيرها. يمكن مشاهدة الفيديو للتوضيح. ولتفعيل هذه الفلاتر يجب أن تكون المتغيرات التالية تساوي `true`:

```

NANO2_PROPOSALS_IS_ALLOW_FILTER_BACKEND_USER = true

# إظهار فلاتر المقترحات والبلاغات في واجهة مستخدمي الفرونت
NANO2_PROPOSALS_IS_ALLOW_FILTER_FRONTEND_USER = false

# إظهار فلاتر المقترحات والبلاغات في واجهة إدارة المتاجر
NANO2_PROPOSALS_IS_ALLOW_FILTER_DEPARTMENT = false
```

مثلاً، لو قمنا بتطبيق إجراء (إنقاص السعر) بخصم بنسبة 20% على الأصناف، ومن ثم أردنا إرجاع الأسعار إلى ما كانت عليه. في هذه الحالة، سنقوم بعمل إجراء آخر باسم (زيادة في السعر) وفي حقل نسبة أو مبلغ ثابت سنقوم باختيار الخيار التالي (إعادة احتساب السعر باستخدام النسبة السابقة). 

يرجى مشاهدة الفيديو لتوضيح آلية عمل الإجراءات الجماعية للأصناف وكيفية عملها.

## من تاريخ 2024-11-28 إلى تاريخ 2024-12-13 بشكل متقطع
**Support Auto Subscribe Topics to FCM in Backend**

إمكانية إشراك المستخدمين في مواضيع إشعارات الفايربيس من خلال الباك اند، حيث تم برمجة كلاس لعمل ذلك وتم عمل دوال لاختبار هذا الكلاس. بحيث أصبح بالإمكان تفعيل خاصية الاشتراك التلقائي في المواضيع الرئيسية، مثلاً مستخدمين الفرونت اند يتم إشراكهم في المواضيع التالية تلقائياً عند تسجيل التوكين الخاصة بهم.

**أسماء المواضيع هي كالتالي لمستخدمين الفرونت اند:**
```
all,frontend,client
```

**وأسماء المواضيع لمستخدمين الباك اند هي كالتالي:**
```
all,backend
```

لتفعيل خاصية إشراك المستخدمين تلقائياً في المواضيع عند تسجيل التوكين الخاصة بهم في النظام، يجب تفعيل الخيار التالي في ملف الإعدادات:
```
# تسجيل الاشتراكات بشكل تلقائي عند إضافة التوكن 
NANO_DEVICES_IS_AUTO_SUBSCRIBE_TOPICS = true
```

## 2024-12-14
**Support Exclude Step_fields in Config_data Tags_type API**

تحديثات في جزء الـ API الخاص بجلب الفئات الرئيسية:
```
api/v1/tags/types
```

حيث أصبح بالإمكان التحكم بإرجاع حقل إعدادات حقول إضافة الإعلان المسمى بـ:
```
config_data[step_fields]
```

حيث إنه سابقاً كان يتم إرجاع هذه الإعدادات حتى في حالة تضمين العلاقة ضمن بيانات الأصناف، وهذا ليس له داعٍ، لأن هذه الإعدادات تستخدم عند إضافة الأصناف أو الإعلانات فقط. لذلك تم عمل آلية للتحكم بمواضع إرجاع هذه الإعدادات مع إمكانية فرض إرجاع هذه الإعدادات.

لإيقاف عرض هذه الإعدادات بشكل دائم افتراضياً، نقوم بتغيير قيمة الإعداد التالي إلى `false`:
```
NANO_TAGSAPI_TYPES_IS_CONFIG_DATA_STEP_FIELDS = 1
```

أيضاً تم كتابة التوثيق في جزء:
```
api/v1/tags/types
```

**ملاحظة: للتوضيح أكثر عن إعدادات الحقول، انظر المثال رقم 1.1.2 ورقم 1.1.3 ورقم 1.1.4.**

**يرجى مشاهدة الفيديوهات لمعرفة التغييرات التي أحدثها هذا التحديث.**

- **إيقاف عرض المراجعات السيئة في الـ API واستبدالها بعبارة معينة**

  ```
  NANO_REVIEWS_REPLACE_NOT_POSITIVE_CONTENT = 'تم إخفاء المراجعة مؤقتاً حتى يتم مراجعتها'
  ```

يمكن مشاهدة الفيديو والصور للتوضيح.

## 2024-12-15 إلى 2024-12-22
**Support Secret Preferences User Data and User Profile**

استكمال لآلية التحكم بملف البروفايل الخاص بالمستخدمين، بحيث يمكن لكل مستخدم التحكم بخصائص بروفايله كإظهار الاسم والعنوان والجنس وإظهار رقم الهاتف والبريد وبيانات التواصل وغيرها. وذلك لضمان الحفاظ على بيانات المستخدمين وعدم السماح بانتهاك خصوصيتهم، والسماح لهم بتحديد المعلومات التي يرغبون بمشاركتها فقط. حيث تم تطبيق هذه الآلية بشكل احترافي في الـ API، وتم عمل إعدادات لذلك بغرض تفعيل أو إيقاف هذه الآلية. وتم اعتماد منطق 
تفضيلات المستخدم العامة، والذي يعتبر جزءاً من تفضيلات المستخدم التي تحدثنا عنها في تحديثات سابقة، والآن تم إدخالها بشكل عملي في برمجة واجهات التطبيقات.

**التفضيلات العامة للمستخدم تتيح التحكم بالحقول والخصائص الظاهرة في ملف البروفايل وغيرها.**

**لتفعيل خاصية التحكم بملف البروفايل عن طريق التفضيلات يجب أن تكون الإعدادات التالية مفعلة:**

- **السماح بتفعيل آلية حماية بيانات المستخدمين**
  ```
  NANO_AUTHAPI_USER_DATA_IS_ALLOW_SECRET = true
  ```

- **فحص بيانات المستخدم بناءً على إعداداته للتحكم بعرض بياناته في الملف الشخصي**
  ```
  NANO_AUTHAPI_USER_DATA_IS_SECRET_CHECK = true
  ```

**للاطلاع على الـ API الخاص بجلب التفضيلات العامة للمستخدم، انظر الـ API التالي:**
```
GET /api/v1/me/preferences
```

**للاطلاع على الـ API الخاص بتعديل التفضيلات العامة للمستخدم، انظر الـ API التالي:**
```
PUT /api/v1/me/preferences
```
