## 2025-04-03
**Support Hot Food Order Type**

إضافة نوع طلب جديد إلى جزء إدارة الطلبات باسم `hot_food` خاص ببعض الوجبات، حيث يتيح هذا النوع لأصحاب المطاعم عرض الوجبات المتوفرة لديهم بعروض زهيدة بدلاً من رميها نهاية الدوام. ويمكن لأي شخص تصفح هذه الوجبات وشرائها بأسعار مناسبة ورخيصة. وبهذه الخطوة نكون وفرنا وجبات بأسعار رخيصة وقللنا من هدر وتلف الطعام الذي لم يتمكن أصحاب المطاعم من بيعه.

- يمكن التحكم بكافة إعدادات وخصائص هذا النوع من إعدادات نوع الطلب.
- لتفعيل هذا النوع يجب أن يكون الخيار التالي مفعل في الإعدادات علماً أن القيمة الافتراضية هي عدم تفعيل:

```plaintext
# لتفعيل نوع الطلب وجبات ساخنة 
NANO_ORDERS_TYPES_HOT_FOOD = true
```

## 2025-04-08
**Support Filter `is_open_opening` and `is_close_opening` in Departments API Version 2**

تحديث الـ API الخاص بالمتاجر والفروع:

```plaintext
GET /api/basic/departments
```

حيث تم إضافة فلاتر خاصة بجلب المحلات أو الفروع بحسب أوقات عملها (أي المفتوحة أو المغلقة). لجلب المحلات المفتوحة في الوقت الحالي نقوم بتمرير المتغير التالي بالقيمة واحد:

```plaintext
is_open_opening = 1
```

لجلب المحلات المغلقة في الوقت الحالي نقوم بتمرير المتغير التالي بالقيمة واحد:

```plaintext
is_close_opening = 1
```

يمكن أيضاً تمرير التاريخ المراد فحص المحلات فيه ضمن المتغير `open_at`.

- يمكن أيضاً التحكم بالقيم الافتراضية لتطبيق هذه الفلاتر من خلال الإعدادات التالية علماً أن القيمة الافتراضية لهذه الإعدادات هي عدم تفعيل (`false`):

```plaintext
# القيمة الافتراضية لتطبيق فلتر السجلات المفتوحة في الوقت الحالي فقط 
NANO_BASICAPI_DEPARTMENTS_DEFAULT_IS_OPEN_OPENING = false
          
# القيمة الافتراضية لتطبيق فلتر السجلات المغلقة في الوقت الحالي فقط 
NANO_BASICAPI_DEPARTMENTS_DEFAULT_IS_CLOSE_OPENING = false
```

وتم تحديث التوثيق الخاص بهذا الجزء مع إضافة عدة أمثلة توضيحية. يمكن مشاهدة الفيديو والصور للتوضيح.

```plaintext
# إرسال كود التحقق بشكل تلقائي عبر البريد عند استخدام دالة إرسال كود التحقق عن طريق رسالة عبر الرقم

```
